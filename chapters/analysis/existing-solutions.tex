\section{Existující řešení}
Následující podkapitola je věnována popisu existujících řešení, která umožňují sledovat a případně i porovnávat výkonnost vývojářů. Mezi tato řešení patří i samotné nástroje GitLab a GitHub, které nabízejí bezplatné zobrazení základních statistik o hostovaných repozitářích. Kromě těchto dvou nástrojů budou v následujících sekcích představena i řešení z komerční sféry, která mi přišla nejrozšířenější nebo nějakým způsobem zajímavá.

\subsection{GitLab}
Nástroj GitLab nabízí několik vizualizací, díky nimž je možné získat přehled o prováděných aktivitách na projektové, skupinové a instanční úrovni. Projektová úroveň odpovídá jednomu konkrétnímu repozitáři, skupinová zahrnuje jeden nebo více souvisejících projektů a instační obsahuje všechny projekty v daném systému.

V bezplatné variantě jsou avšak k dispozici pouze základní statistiky. Jedná se například o informace o využívaných programovacích jazycích, o počtu provedených commitů nebo o době trvání a úspěšnosti doběhnutí pipeline. \cite{GitLab-analytics} Zajímavější statistiky jsou k dispozici u placených plánů, jejichž cena začíná na 29 dolarech za uživatele na měsíc. \cite{GitLab-pricing} Ty následně zahrnují například statistiky o merge requestech a code review. U merge requestů se jedná o (průměrný) čas do mergnutí, počet mergnutých merge requestů za zvolené časové období a například počet commitů a změněných řádků u jednotlivých merge requestů. \cite{GitLab-merge-requests} Statistiky code review obsahují mimo jiné délku review procesu (čas od přidání prvního komentáře jinou osobou než autorem merge requestu), počet komentářů nebo seznam approvers. \cite{GitLab-code-review}

\subsection{GitHub}
Platforma GitHub stejně jako nástroj GitLab nabízí několik vizualizací o repozitářích, které jsou na ní hostované. Mezi základní přehledy patří statistiky o přispěvatelích, o počtu přidaných a smazaných řádcích zdrojového kódu nebo například dashboard Pulse. Ten obsahuje informace o počtu otevřených (uzavřených) pull requestů a issues za zvolené časové období společně s commit aktivitou. \cite{GitHub-graphs, GitHub-pulse} Většina těchto přehledů je však dostupná pouze u veřejných repozitářů nebo jsou součástí placených plánů. \cite{GitHub-graphs}

Kromě těchto předpřipravených přehledů GitHub umožňuje vytvářet své vlastní vizualizace v rámci GitHub Projects. GitHub Projects je přizpůsobitelný přehled úkolů a harmonogram, který díky propojení s issues a pull requesty umožňuje efektivní plánování a kontrolu provedené práce. Toho je docíleno možností vytvářet různé pohledy, v nichž je možné filtrovat, řadit a seskupovat issues a pull requesty. \cite{GitHub-projects}

\subsection{Swarmia}
Swarmia je jedním z pokročilých nástrojů pro sledování aktivity a produktivity vývojářů. Data s aktivitami vývojářů jsou primárně získávány z platformy GitHub nebo ze self-hosted GitHub Enterprise Serveru. Díky podporovaným integracím s nástroji pro řízení projektů a evidenci tasků Jira a Linear, dokáže Swarmia poskytovat podklady i pro bussiness rozhodnotí o prioritách, lidech a procesech organizace. \cite{Swarmia-Integrations, Swarmia-business-outcomes}

Jedním z přehledů o projektu, který Swarmia poskytuje, je rozdělení investic na základě typu úkolu. Každému úkolu je přiřazena jedna z následujících pěti kategorií -- nové vlastnosti, vylepšování/úprava existujících funkcionalit, produktivita\footnote{Do kategorie produktivita se zahrnují převážně úpravy podporující produktivitu vývojářů a vylepšení infrastruktury umožňující budoucí růst projektu.}, údržba\footnote{Kategorie údržba se přiřazuje úkolům, které jsou nezbytné pro chod projektu. Příkladem může být oprava chyb, aktualizace závislostí nebo rutinní úkoly.} a nezařazené. Díky tomuto rozdělení je možné sledovat a pokusit se nalézt optimální balanc mezi novými investicemi a údržbou existujících funkcionalit. \cite{Swarmia-investment-balance}

Pro vyhodnocení samotné produktivity vývojářů využívá Swarmia SPACE framework a DORA metriky, které byly představeny v předchozí podkapitole. Využívané metriky zahrnují například dobu cyklu pull requestu, kterou Swarmia definuje jako součet času od prvního commitu nebo vytvoření pull requestu do času prvního vyžádání review, času od prvního vyžádání review do posledního schválení pull requestu a času od posledního schválení pull requestu do samotného mergnutí pull requestu. \cite{Swarmia-pull-request-cycle} Mezi další metriky patří flow efektivita, čas do nasazení nebo například velikost změn, která je definována jako počet upravených řádků v rámci jednoho pull requestu bez automaticky generovaných souborů. \cite{Swarmia-metrics, Swarmia-batch-size}
